## 1. 基本原则

参考文献格式不预设为AMA、GB/T 7714或任何其他体例。唯一可用于投稿定稿的规则，是目标期刊当前有效的官方投稿须知、投稿系统说明或该刊提供的正式模板。

“北大核心”“中文核心”不是统一的引文格式。即使两种格式都采用数字顺序制，其上标、括号、标点位置、作者列名、文献类型标识、DOI和英文大小写规则也可能不同。

目标期刊未明确，或官方要求无法核实时，只能制作标为“非投稿定稿”的工作稿，并保留规范化内部引文标记；不得自行宣称符合期刊要求。

## 2. 官方要求的核验顺序

联网核验具有时效性的投稿要求，并记录页面题名、URL、发布日期或版本、访问日期和关键原文。来源优先级为：

1. 目标期刊官方网站或官方投稿系统中当前有效的作者指南。
2. 期刊官网下载的最新投稿模板、格式说明或参考文献示例。
3. 编辑部可核验的正式通知或用户提供的当前书面要求。
4. 期刊近期正式刊文，只用于补足官方指南未说明的版式细节，不用于推翻明确的官方规则。
5. 数据库说明、转载投稿须知和第三方经验帖仅作发现线索。

若官方页面之间冲突，记录冲突和各自日期，优先采用明确标注为当前有效且版本较新的规则；仍不能判断时标为“待编辑部确认”，不得凭经验选择。

## 3. 冻结期刊格式画像

正式插入引文前建立 `JOURNAL_CITATION_PROFILE`，至少填写：

| 字段 | 必填内容 |
|---|---|
| 期刊与稿件类型 | 期刊全名、栏目或文章类型 |
| 核验来源 | 官方页面/模板题名、URL、版本或发布日期、访问日期 |
| 引文体系 | 数字顺序制、著者—年份制、脚注/尾注或期刊自定义 |
| 正文标记 | 方括号/圆括号/裸数字，基线/上标，是否保留括号 |
| 标点位置 | 引文标记置于句号、逗号或引号之前/之后 |
| 多篇合引 | 连续编号压缩方式，非连续编号分隔符，是否允许区间 |
| 表图与脚注 | 是否计入首次引用顺序及其标注方式 |
| 文后排序 | 首次出现顺序、作者字母顺序或其他规则 |
| 序号样式 | `[1]`、`1.`、`1`、无序号或期刊自定义 |
| 著录标准 | 期刊自定义、GB/T 7714指定版本、AMA指定版本等 |
| 作者规则 | 全列人数、超过人数后的省略方式、中外作者写法 |
| 刊名与题名 | 全称/缩写、英文大小写、双语著录要求 |
| 卷期页码 | 年卷期页、文章号、增刊和预发表文章写法 |
| DOI与URL | 是否必填、呈现形式、访问日期要求 |
| 专著与古籍 | 版次、整理/校注者、出版地、出版社、年份和引文页码 |
| 版式 | 字体字号、悬挂缩进、行距、段间距和标点全半角 |
| 数量与时效 | 篇数上限/下限、近年文献比例等明确要求 |
| 未决项 | 官方指南未说明或相互冲突的项目 |

画像中的每项必须能回到核验来源。期刊未规定的字段写“未规定”，不能把常见做法冒充期刊要求。

## 4. 正文引文插入

1. 每条引文紧邻其支持的事实或判断，不能只在整段末尾集中放置而造成支持对象不清。
2. 直接引文除文献号外，按期刊要求补页码；期刊未说明时仍在证据台账保存精确页码。
3. 正文、摘要、表格、图注和脚注是否参与编号，以期刊画像为准。
4. 数字顺序制按期刊规定的首次出现范围编号；删改正文后必须整体重排。
5. 连续和非连续文献的连接符、分隔符及空格完全按期刊示例，不自行混用中英文标点。
6. 上标、括号和标点位置在Word成稿中检查，不能只检查纯文本编号。
7. 著者—年份制必须核对作者拼写、年份、同年文献后缀及文后排序；不得运行数字顺序重排脚本。
8. 不用一个引文笼统支持同一句中的多个不同结论；必要时拆句并分别定位文献。

Markdown工作稿可暂用 `[1]` 作为数字顺序制的内部标记，但这不是投稿格式。生成Word时必须依据画像转换为期刊要求的最终标记。

## 5. 文后参考文献著录

逐条对照原文首页、数据库正式记录、出版社版权页或官方网页，不从其他论文的参考文献表直接复制未经核验的书目信息。

重点核对：

- 作者姓名、作者数量和省略规则。
- 题名、文献类型标识及中英文大小写。
- 期刊全名或规范缩写。
- 年、卷、期、起止页码或文章号。
- 版本、出版地、出版社、出版年和引文页码。
- DOI是否真实解析并指向该文献；不得根据格式猜测DOI。
- 网络资源的责任机构、页面题名、更新日期、访问日期和URL。
- 非英文文献是否需要英译题名、语言标识或中英文双列。
- 全角/半角标点、空格、悬挂缩进和字体字号。

期刊引用GB/T 7714、AMA或其他标准时，先确认指定版本及该刊是否有覆盖性自定义规则；期刊给出的明确示例优先于标准中的一般示例。

## 6. 古籍、本草与特殊来源

- 论文正文可说明原著时代与作者，文后书目著录实际核读的影印本、校勘本或整理本，不把古代成书年代伪作现代出版年。
- 古籍直接引文记录卷次、篇名、现代版本页码和PDF页码；最终呈现哪些字段按期刊要求，完整定位仍保留在证据台账。
- 据汇编本转引而未见原书时明确标注转引关系，不把汇编条文改写成“据原书核得”。
- 药典、标准、法规和指南按期刊对责任机构、版本、发布机关、年份、编号和页码的要求著录。
- 学位论文、会议摘要、预印本、数据库记录和网页能否引用，以目标期刊规则为准；不满足要求时更换为可接受来源或披露缺口。
- 岐黄之术等聚合页面原则上只作发现来源；除非文章研究对象就是该网页，否则不替代原书进入核心参考文献。

## 7. 数字顺序制自动检查

仅当画像确认采用数字顺序制时，才运行：

```bash
python scripts/audit_numeric_citations.py manuscript.md --write
python scripts/audit_numeric_citations.py manuscript.md
```

脚本检查内部 `[n]` 标记的首次出现顺序、未定义引用、未被引用书目和重复编号。它不决定上标、括号、标点位置，也不校正每条书目的期刊著录格式。

生成Word时显式传入期刊画像，例如：

```bash
python scripts/build_bencao_docx.py manuscript.md manuscript.docx \
  --citation-style superscript-bracket \
  --reference-label-style bracket
```

可用正文数字标记：`superscript-number`、`superscript-bracket`、`inline-bracket`、`inline-parenthesis`。文后序号可用 `dot` 或 `bracket`。选择必须来自期刊画像，不得因脚本默认值而决定格式。

## 8. 著者—年份制检查

若期刊采用著者—年份制：

1. 正文直接写入期刊要求的作者和年份形式。
2. 文后书目按期刊规定的作者字母、中文拼音或其他顺序排列。
3. 同一作者同一年文献按期刊规则添加a、b、c等后缀。
4. 运行Word脚本时使用 `--citation-style author-date --reference-label-style none`，脚本会拒绝残留数字 `[n]` 标记。
5. 通过人工或适用的文献管理软件逐条核对，不运行数字顺序重排脚本。

若期刊采用脚注、尾注或自定义引文系统，停用数字重排和上述Word引文转换选项，改用文档技能按期刊模板创建真实注释结构；不得用手工上标文本伪装Word脚注。

## 9. 证据匹配与真实性审计

- 每个引用数字或著者—年份标记均有对应书目。
- 每条文后书目至少在允许引用的稿件区域出现一次。
- 无重复书目、重复DOI或同一来源的多种冲突写法。
- 引文所在句确由该来源支持，且结论强度不超过证据等级。
- 不用综述替代能够获得的原始研究，不用二手汇编冒充原书。
- 不用复方、提取物或单体研究支持单味饮片的独立疗效。
- 引用撤稿、勘误或存在重大争议的研究时说明状态和使用边界。
- 无法取得全文或核实书目信息的来源标为“待核”，不得进入投稿定稿。

## 10. 修改后终检与阻断条件

作者修改后重新执行：期刊画像复核、引文重排、书目信息核对、证据匹配、Word最终呈现和逐页检查。

出现以下任一情况，不得声称“参考文献已符合期刊要求”：

- 目标期刊或稿件类型未明确。
- 当前官方指南、模板或有效版本未核验。
- 引文体系、正文标记或文后排序仍有冲突。
- 存在未定义引用、未引用书目、重复编号或正文与文后不对应。
- DOI、页码、作者或出版信息未经核实。
- Word中的上标、括号、标点位置、字体或悬挂缩进未逐页检查。

最终审计报告应列出期刊画像来源、自动检查结果、人工抽查结果、全部未决项和是否通过投稿格式门禁。
