# 中文本草考证医学综述

## 总则

把古籍原文、版本校证、药典标准、现代实验和临床证据分层处理。优先保证可核查性，不以资料数量替代证据质量，不把汇编网页冒充原书，不把复方疗效反推为单味药疗效。

需要处理PDF时同时使用PDF技能；需要生成或修改Word时同时使用文档技能；涉及一般医学综述证据分层时同时使用 `medical-cn-review`。

## 1. 冻结任务约束

开始前从用户材料中提取并固定以下要求：

- 研究药物、目标期刊和文章类型。
- 作为写作范式的样文，只学习结构与证据组织，不在正文写“按照某文范式”。
- 允许的图表数量。未另行说明时只保留两张核心表，不增设图3、图4或重复性附表。
- 引文格式、参考文献语言、字体字号和交付文件格式。
- 用户提供的药典报告、重写报告、初稿和古籍扫描件的优先级。

编辑现有Word时保留其Times New Roman字体和字号，除非用户明确要求改变。不要因重新生成文档而无意覆盖既有样式。

## 2. 建立证据台账

先建立“待证问题-证据-页码-结论-限制”台账，再写正文。按以下层级记录：

1. 原书影印本、可靠校勘本、现行药典和官方监管文件。
2. 正式出版的古籍整理本、汇编本和本草考证专著。
3. 中文核心期刊考证论文、药材质量研究、系统评价和指南共识。
4. 动物、细胞、网络药理学及其他基础研究。
5. 聚合网页、岐黄类书目站和百科页面，只用于发现线索，不作为可替代原书的最终证据。

详细规则见 [references/evidence-and-writing.md](references/evidence-and-writing.md)。

## 3. 校证扫描版古籍PDF

按以下顺序处理：

1. 检查PDF是否有文字层；无文字层时按扫描件处理。
2. OCR目录，确定目标药物的书内印刷页码。
3. 通过页眉、页脚或相邻条目确定“PDF页码-书内页码”偏移。
4. 对目标页及前后续页做高分辨率渲染和OCR。
5. 同时查看原图，逐句修正OCR中的异体字、误分词和标点。
6. 引用直接引文时记录书内印刷页码；只有PDF页码而无印刷页码时明确标为PDF页码。
7. PDF临时缓存失效时停止页码级补写，要求用户重新上传或移至稳定目录；不得凭网页内容虚构纸本页码。

若同一经典存在“味苦涩”与“味苦，寒”等异文，分别记录版本来源并在正文保留差异。

## 4. 组织论文

推荐正文结构：

1. 摘要与关键词。
2. 引言：临床与药典背景、既往考证、未解决问题、资料来源和方法。
3. 名称考证。
4. 基原与药用部位考证。
5. 性味、归经及毒性考证。
6. 功效应用、采收炮制及现代标准。
7. 讨论与结论：证据边界、争议、现代病理药理衔接及未来方向。
8. 参考文献。

默认只设置以下两张表：

- 表1：历代药物名称表。字段为年代、名称、入药部位、出处。
- 表2：历代性味、归经、毒性表。字段为出处、入药部位、四气、五味、归经、毒性。

古籍未明确的字段写“不详”；医家理论推演写“医家解释”，不得伪装成原书固定字段。名称表区分正名、异名、俗名、商品地域名和药典习称。

## 5. 写作与推理边界

- 每节围绕一个待证问题展开，不堆砌条文。
- 先陈述原文或可靠释义，再作跨时代比较，最后给出有限结论。
- 把单味药、饮片质量、代表成分和含药复方分开评价。
- 不把“肝胆湿热”“肝火上炎”等传统术语与单一现代疾病或通路直接等同。
- 动物、细胞和网络药理学结果只能支持机制假说，不能替代临床疗效。
- “无毒”不等于长期、大量或不辨证使用安全；同时讨论剂量、疗程、体质、炮制和配伍。
- 保留《本经逢原》“小毒”、张锡纯“胃家正药”等少数意见，并标明其非历代共识。
- 正文不得出现“本文按照某样文写作”“为了压缩图表”等过程说明。

## 6. AMA顺序编码

正文、表格和图注共同参与首次引用排序。使用方括号编码维护Markdown，Word中转换为上标。专著直接引文尽量给出页码；网页写明发布日期、访问日期和URL。

不要为达到“约50篇”而填充无关文献。每条参考文献必须支持正文中的具体句子，且正文引用、表格来源和文末书目信息三者一致。

修改后运行：

```bash
python scripts/reorder_ama.py manuscript.md --write
python scripts/reorder_ama.py manuscript.md
```

详细规则见 [references/ama-and-audit.md](references/ama-and-audit.md)。

## 7. 生成投稿Word

编辑既有DOCX时优先原位修改和保留样式。由Markdown新建时可运行：

```bash
python scripts/build_bencao_docx.py manuscript.md manuscript.docx
```

默认样式：

- 中文正文：宋体，10.5磅，1.25倍行距。
- 英文、拉丁学名和拉丁字母：Times New Roman，10.5磅；不要擅自改变。
- 一级标题：黑体，13磅；主标题：黑体，18磅。
- 参考文献：宋体，9磅，悬挂缩进。
- 引文序号：上标，不保留方括号。
- 表格：三线表；表1纵向，表2信息较宽时横向；跨页重复表头且禁止整行拆分。
- 不使用蓝色标题、灰底全框线或额外装饰。

详细样式与检查项见 [references/word-and-qa.md](references/word-and-qa.md)。

## 8. 投稿前核查

必须完成以下检查后再交付：

1. AMA首次引用顺序连续，无漏引、无未定义引用、无未被引用书目。
2. 全文只有用户允许的图表；默认恰好两张表。
3. 表格每行均能追溯到具体来源，未载字段保持“不详”。
4. 古籍直接引文与原页图像一致，页码为书内页码而非猜测。
5. 现代药理、病理和临床证据没有越级外推。
6. 中文文献语言、英文术语和拉丁学名符合用户要求。
7. DOCX逐页渲染，检查表头重复、横竖版切换、字体替换、文字截断和空白异常。
8. 最终说明已完成内容、验证结果及仍需原件核对的限制。
