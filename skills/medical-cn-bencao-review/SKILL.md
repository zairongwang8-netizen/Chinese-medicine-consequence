# 可复制的中文本草考证工作流

## 总则

把每次任务视为“目标药物变量 + 来源配置 + 证据台账 + 问题轴/时间轴综合 + 引用与版式审计”，不得把某一药物的结论、书目命中情况或网页地址硬编码到下一项任务。

古籍原文、版本校证、法定标准、现代实验和临床证据必须分层处理。优先保证可核查性，不以资料数量替代证据质量，不把聚合网页冒充原书，不把复方或单体成分研究反推为单味饮片疗效。

需要处理PDF时同时使用PDF技能；需要生成或修改Word时同时使用文档技能；涉及一般医学综述证据分层时同时使用 `medical-cn-review`。

本技能不在公开仓库分发仍受版权保护的整本扫描书。用户合法提供的PDF只在当前任务环境内读取；技能仓库仅保存工作流、书目线索、校证规则和工具脚本。

## 1. 冻结任务变量

写作和检索前建立任务配置；缺失项从用户材料中推断，并把推断标为“待确认”。

| 变量 | 内容 |
|---|---|
| `TARGET_HERB` | 目标药物法定名或研究正名 |
| `TARGET_SYNONYMS` | 异名、古名、俗名、地域名、商品名及可能混用品名 |
| `TARGET_SCOPE` | 名称、基原、药用部位、产地、采收、炮制、药性、功效、毒性等待证范围 |
| `TIME_RANGE` | 考证起止时代，默认先秦至现行药典 |
| `CORE_SOURCES` | 用户合法提供的原书、校勘本、汇编本、样文和报告 |
| `OFFICIAL_STANDARDS` | 现行及关键历史版药典、部颁标准、地方标准或监管文件 |
| `DISCOVERY_SOURCES` | 岐黄之术、目录数据库、图书馆目录和在线镜像，仅用于发现线索 |
| `MODERN_DATABASES` | 用户可访问的中文数据库、PubMed及其他学术数据库 |
| `TARGET_JOURNAL` | 目标期刊、文章类型、字数和格式要求 |
| `TABLE_POLICY` | 默认两张核心表；用户批准后方可调整数量或字段 |
| `CITATION_POLICY` | 默认AMA顺序编码；同时记录语言、页码和网页访问日期要求 |
| `REVIEW_PROFILE` | 目标期刊审稿侧重点；默认执行中文核心本草类六阶段模拟审稿 |
| `OUTPUTS` | 证据台账、初稿、DOCX、审计报告等交付物 |

不得把样文中的药名、专属判断、表格行或参考文献直接迁移到目标药物。样文只用于学习问题设置、时间组织、论证密度和版式。

## 2. 配置来源并通过来源门禁

先读取 [references/source-first-protocol.md](references/source-first-protocol.md)，建立本任务来源矩阵。按以下顺序推进：

1. 清点用户附件和工作区中的合法PDF、DOCX、图片、药典报告和既有初稿，记录文件状态与用途。
2. 以 `TARGET_HERB` 和 `TARGET_SYNONYMS` 检索每一本配置的核心书目；某书未收录时记录“未命中、检索词、检索范围”，然后继续下一来源，不得强行套用。
3. 命中扫描版PDF后，读取目录和目标条目的全部连续页，记录PDF页码、书内印刷页码、章节边界和相邻条目。
4. OCR后必须回看原页图像。未经原图复核的文字标为“仅据OCR”或“待校”，不得作为直接引文。
5. 检索岐黄之术 `https://qihuangzhishu.com` 时，使用目标药正名、异名和候选古籍书名组合检索；新增条文进入“待回核队列”。
6. 聚合站、目录页和在线镜像只能发现书目、异名或条文位置；核心结论须回核原书影印、可靠校勘本、正式出版物或官方标准。
7. 关键来源缺失时可以继续制作缺口清单和条件性初稿，但必须明确限制，不得虚构原文、版本、页码或“全面考证”结论。

## 3. 建立证据台账

先建台账，再填表和写正文。每条证据至少包含：

| 字段 | 填写要求 |
|---|---|
| 待证问题 | 该条证据回答什么问题 |
| 名称与版本 | 药名/异名，书名、作者、版本、出版信息 |
| 来源类型与层级 | 原书、校勘本、标准、汇编、论文、网页线索及A-E层级 |
| 定位 | 书内页码、PDF页码或URL分别记录 |
| 原文或释义 | 短引文或忠实释义；不得混合个人推断 |
| 证据用途 | 名称、基原、部位、产地、炮制、药性、功效、毒性等 |
| 校证状态 | 已核原图、可靠校勘本、仅据OCR、据转引、待回核、未命中 |
| 版本差异 | 异文、字段冲突和可能原因 |
| 引用状态 | 拟引用位置、临时编号、AMA核对状态 |
| 限制与备注 | 证据边界、缺页、版本不明或不可访问情况 |

证据层级：

1. A：原书影印、可靠校勘本、现行药典和官方监管文件。
2. B：正式出版的古籍整理本、汇编本、本草考证专著。
3. C：中文核心考证论文、质量研究、系统评价、指南或共识。
4. D：动物、细胞、网络药理学及其他基础研究。
5. E：聚合网页、在线镜像、百科和搜索摘要，仅作线索。

详细规则见 [references/evidence-and-writing.md](references/evidence-and-writing.md)。

## 4. 按问题轴和时间轴综合

不要按“找到一本书就抄一段”的顺序写作。先为每个问题建立时间链：

1. 名称何时首次出现，正名、异名和俗名如何分化或合流？
2. 所指植物、动物、矿物或加工品是否发生替代、混用或地域分化？
3. 药用部位、产地、采收、修治和炮制认识如何变化？
4. 四气、五味、归经、升降浮沉和毒性判断何时形成，是否存在异文？
5. 功效主治和医家使用评价如何扩展，哪些是原书事实，哪些是后世解释？
6. 现行药典如何收束历史多样性，尚存哪些鉴别、质量或安全问题？
7. 现代药理、病理和临床证据能解释哪些传统认识，哪些仍只是研究假说？

每节采用“待证问题 -> 早期证据 -> 后世演变 -> 版本/学术争议 -> 有限结论”的结构。所有跨时代结论必须能回到台账中的多条证据，不能由单条文献外推。

## 5. 组织论文和核心表格

推荐正文结构：

1. 摘要与关键词。
2. 引言：研究价值、既往考证、未解决问题、资料来源与方法。
3. 名称考证。
4. 基原与药用部位考证。
5. 性味、归经及毒性考证。
6. 功效应用、医家评价、采收炮制及现代标准。
7. 讨论与结论：争议、证据边界、现代病理药理衔接及未来研究方向。
8. 参考文献。

默认仅设置两张核心证据表：

- 表1“历代药物名称表”：年代/时代、名称或异名、入药部位、出处及引文。
- 表2“历代性味、归经、毒性表”：出处、入药部位、四气、五味、归经、毒性及引文。

若目标药物的核心争议是基原、产地、炮制或混伪品，可提出调整字段或表题；说明调整所回答的研究问题并取得用户同意。不得为展示资料而任意增加图表。古籍未明确的字段写“不详”；推演内容标为“医家解释”或“作者分析”，不得伪装成原书字段。

## 6. 写作与推理边界

- 每节围绕一个待证问题，不堆砌条文。
- 先陈述已校证原文或可靠释义，再比较时代与版本，最后给出有限结论。
- 区分法定药材、饮片、提取物、代表成分、混用品和含药复方。
- 不把传统证候、脏腑或治法术语与单一现代疾病、病理过程或分子通路直接等同。
- 动物、细胞和网络药理学结果只支持机制假说，不能替代临床疗效证据。
- 含目标药复方的疗效不能写成该单味药的独立疗效。
- “无毒”或药典未标毒性不等于长期、大量或不辨证使用安全；同时评价基原、剂量、疗程、体质、炮制、配伍和不良反应归因。
- 少数医家意见必须给出原始出处、时代语境和证据地位，不得写成历代共识。
- 正文不得出现“按照某样文写作”“为了压缩图表”等过程说明。

## 7. AMA顺序编码和书目审计

正文、表格、图注和脚注共同参与首次引用排序。Markdown阶段使用方括号编码，Word中转换为上标。古籍和专著直接引文尽量给出版本与页码；网页写明页面题名、发布或更新日期、访问日期和URL。

不要为达到固定篇数填充无关文献。每条参考文献必须支持具体句子或表格行，且“证据台账 -> 正文/表格引用 -> 文末书目”三者一致。

修改后运行：

```bash
python scripts/reorder_ama.py manuscript.md --write
python scripts/reorder_ama.py manuscript.md
```

详细规则见 [references/ama-and-audit.md](references/ama-and-audit.md)。

## 8. 强制执行多阶段模拟审稿

内部工作稿形成后、向用户提交任何论文初稿前，必须读取 [references/reviewer-and-deai.md](references/reviewer-and-deai.md)，依次完成“编辑初审 -> 三名同行独立评审 -> 交叉互审 -> 模拟主编终审 -> 作者修改 -> 原审稿人复审”。不得在未形成独立审稿意见前直接做表面润色。

三名同行分别负责本草文献与名物考证、中药学与研究方法、期刊结构与学术表达。独立评审阶段使用同一冻结输入包且互不可见；完成后才允许交叉互审。环境支持子代理时优先隔离执行，否则以三个独立角色分轮落盘，不得让后续角色附和前一份意见。

编辑初审低于75分、存在真实性/数据/伦理/核心方法阻断项，或模拟主编不允许修改时，停止自动改稿并向用户报告缺口。不得用语言润色掩盖根本问题，也不得因评分较高越过阻断项。

可先运行风险标记器：

```bash
python scripts/reviewer_lint.py manuscript.md --output reviewer-language-audit.md
```

脚本只生成线索，不自动修改文章。最终至少形成编辑初审、三份独立审稿意见、交叉互审表、模拟主编终审、作者响应台账和三份复审意见。完成修改后重新运行AMA审计；引用顺序可能因删改而变化。

## 9. 生成和逐页核查Word

编辑既有DOCX时优先原位修改并保留用户指定字体、字号和样式。由Markdown新建时可运行：

```bash
python scripts/build_bencao_docx.py manuscript.md manuscript.docx
```

未指定期刊模板时采用中文核心常用默认值：中文正文宋体10.5磅，西文和拉丁学名Times New Roman 10.5磅，一级标题黑体13磅，主标题黑体18磅，参考文献宋体9磅；引文序号上标；表格使用三线表并跨页重复表头。

生成后必须把DOCX渲染为逐页图像或PDF，从第一页检查到末页。详细规则见 [references/word-and-qa.md](references/word-and-qa.md)。

## 10. 交付门禁

仅在以下项目全部通过或明确披露限制后交付：

1. 任务变量和来源矩阵已冻结，未把上一药物的内容迁入本任务。
2. 每个核心书目均记录“命中/未命中”和检索词；命中扫描件已建立PDF页码与书内页码映射。
3. 直接引文已核原图或可靠校勘本；OCR、转引和网页线索均有状态标记。
4. 岐黄之术等发现来源中的关键线索已回核，未把网页地址写成纸本版本或页码。
5. 两张默认表或经批准的替代表格逐行可追溯，未载字段保持“不详”。
6. AMA首次引用顺序连续，无漏引、未定义引用、重复书目或未被引用书目。
7. 现代药理、病理和临床证据没有越级外推，复方与单味药结论已分开。
8. 编辑初审、三名独立评审、交叉互审、模拟主编终审、作者修改和原审稿人复审均已完成。
9. 所有A级问题已解决；未解决B-D级问题、真实性/伦理/方法阻断项均已披露，且未把受阻稿件称为可投稿稿。
10. 空泛价值判断、模板化衔接和无证据强结论已删除或具体化；锁定数据与直接引文未被误改。
11. 修改后重新完成AMA审计，文稿语言、字数、字体和字号符合已核目标期刊要求或通用配置。
12. DOCX已逐页检查表头、横竖版切换、字体替换、文字截断、URL溢出和异常空白。
13. 交付说明列明全部模拟审稿材料、作者响应、自动检查结果和仍需原件复核的缺口。

## 11. 可复制启动指令

```text
使用 $medical-cn-bencao-review 对【TARGET_HERB】开展本草考证。
异名/混用品：【TARGET_SYNONYMS】
研究范围：【TARGET_SCOPE】
核心资料：【CORE_SOURCES】
目标期刊与字数：【TARGET_JOURNAL】
表格要求：默认两张核心表；如需调整先说明理由并征得同意。
引用与字体：AMA顺序编码；中文和西文字体按期刊或用户要求。
交付：证据台账、问题轴/时间轴修订稿、多阶段模拟审稿包、作者响应台账、参考文献审计、逐页检查后的DOCX。
先报告来源命中情况和关键缺口，再进入写作；不得引用未核验原文或虚构页码。
内部工作稿完成后强制执行编辑初审、三名独立同行评审、交叉互审、模拟主编终审、作者修改及原审稿人复审；完整流程通过前不得称为可投稿初稿。
```
